=== COMMAND 3: Output Redirection (>) ===
Command: echo 'Hello, Output!' > output_test.txt
Hello, Output! This was written via output redirection.
-e 
Observation: The '>' operator redirects stdout (FD 1) from the terminal to a file. The file is created if it doesn't exist, or truncated if it does. This is one of the most used I/O features in Linux shell scripting.
-e 
=== COMMAND 4: Append Redirection (>>) ===
Command: echo 'Second line' >> output_test.txt
Hello, Output! This was written via output redirection.
Second line - appended using >>
-e 
Observation: The '>>' operator appends to a file without truncating existing content. This is useful for log files and cumulative reports where historical data must be preserved.
-e 
=== COMMAND 5: Error Redirection (2>) ===
Command: cat nonexistent_file.txt 2> error_log.txt
Error captured in error_log.txt:
cat: /nonexistent_file_xyz.txt: No such file or directory
-e 
Observation: The '2>' operator redirects stderr (FD 2) to a file. Here, attempting to read a non-existent file produces an error message. Redirecting stderr separately from stdout is essential for clean log files in automated scripts.
-e 
=== COMMAND 6: Combined Stdout + Stderr Redirection (2>&1) ===
Command: (echo 'output line'; cat nonexistent) > combined.txt 2>&1
This is stdout output
cat: /nonexistent_file_xyz.txt: No such file or directory
-e 
Observation: '2>&1' merges stderr into stdout. The '>' then sends the combined stream to a file. This is commonly used in cron jobs and production scripts to capture ALL output (both normal and error) in one log file.

Observation: 'ulimit -a' shows all resource limits for the current shell session. Key limits include: open files (max file descriptors a process can open), stack size, and max processes.
