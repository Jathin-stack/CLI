=== COMMAND 1: Create original file ===
echo 'This is the original project file. Version 1.0' > original_file.txt
This is the original project file. Version 1.0
-e 
Observation: We created 'original_file.txt' using the echo command with output redirection. This file will serve as the source for both hard and symbolic links in this experiment.
-e 
=== COMMAND 2: Create Hard Link ===
ln original_file.txt hard_link.txt
-e 
Observation: 'ln' without any flags creates a hard link. A hard link is another directory entry pointing to the same inode (data block) on disk. Both original_file.txt and hard_link.txt now reference the exact same data.
-e 
=== COMMAND 3: Create Symbolic Link ===
ln -s original_file.txt symlink_file.txt
-e 
Observation: 'ln -s' creates a symbolic (soft) link. A symlink is a special file that stores the path to the target file. Unlike a hard link, it does not share the same inode — it is a pointer/shortcut to the original.
-e
