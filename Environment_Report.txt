1. USERNAME AND GROUPS
---------------------------------------------------------------
Command Used : whoami
Output       : root

Command Used : id
Output       : uid=0(root) gid=0(root) groups=0(root)

Explanation:
The 'whoami' command returns the username of the currently logged-in
user. The 'id' command provides detailed identity information:
- uid=0 : User ID 0 is assigned to root (superuser)
- gid=0 : Primary group is also root (group ID 0)
- groups=0(root) : The user belongs to the root group only

Being 'root' means this account has unrestricted access to all
system commands and files, which is typical in a fresh Linux
environment setup before creating standard user accounts.

2. CURRENT SHELL BEING USED
---------------------------------------------------------------
Command Used : echo $SHELL
Output       : /bin/bash

Explanation:
The $SHELL environment variable stores the path of the default
login shell. '/bin/bash' means the system is using Bash (Bourne
Again Shell), which is the most popular and feature-rich Linux
shell. Bash supports scripting, command history, tab-completion,
and piping — essential tools for a Linux support engineer.

---------------------------------------------------------------
3. CURRENT WORKING DIRECTORY
---------------------------------------------------------------
Command Used : pwd
Output       : /

Explanation:
The 'pwd' (Print Working Directory) command outputs the full
absolute path of the current directory. The output '/' indicates
we are at the root of the filesystem, which is the top-level
directory in the Linux directory tree. All other directories
(/home, /etc, /usr, etc.) branch from here.

Explanation:
'ls -la' lists all files including hidden ones (names starting

- Column 2 : Hard link count
- Column 3 : Owner
- Column 4 : Group
- Column 5 : Size in bytes
- Column 6-8: Last modified date/time
- Column 9 : File/Directory name

The workspace contains hidden config directories (.cache, .config,
.local, .npm) and the 'lab_submission' folder for this assignment.

5. NETWORK CONNECTIVITY VERIFICATION
---------------------------------------------------------------
Command Used : curl -s --max-time 5 -o /dev/null -w "HTTP Status: %{http_code}" https://google.com
Output       : HTTP Status: 403

Explanation:
'curl' is used to test network connectivity by making an HTTP
request to google.com. An HTTP 403 (Forbidden) response, while
not a 200 (OK), confirms that the network stack is functioning
and packets are reaching and returning from the internet.
The 403 is expected in sandboxed/lab environments where direct
HTTP access may be filtered. Network connectivity is VERIFIED.
